Project System Engineering and Analysis - quizsysteem

Beschrijving:

Het hoofddoel van dit project was het digitaliseren van een quiz. Op het moment organiseerde deze vzw de quizmomenten op papier. De doelgroep voor deze quiz waren oudere mensen dus moest het enorm gebruiksvriendelijk zijn. Het quizsysteem moest aan bepaalde eisen voldoen. De quizmaster moest controle hebben over de snelheid van de vragen/rondes. De quizmaster moest ook een scorebord kunnen laten tonen dat telkens geüpdatet wordt. Ook was er een jury waar we rekening mee moesten houden. De jury moest de huidige score per team kunnen aanpassen en ook de juiste antwoorden in de verbetersleutel van te voren doorgeven. De verbetersleutel moest natuurlijk ook aangepast kunnen worden. De antwoorden van de teams moesten via een fysieke knoppendoos doorgestuurd kunnen worden.

Mijn bijdrage:

Waar ik me vooral mee bezig heb gehouden was de front-end en de styling van de front-end. De front-end zijn de webpagina’s dat de Api gaat aanspreken door de interactie van de gebruiker. Dit hebben we gedaan aan de hand van REACT en ik heb deze pagina's stijl gegeven met CSS. Waar ik me ook mee bezig heb gehouden was de fysieke opstellen om de antwoorden door te geven. Dit hebben we gedaan door wat code te schrijven op een Raspberry Pi dat ook de API kan aanspreken. Aan deze Raspberry Pi waren knoppen met Ledjes aan verbonden.

Vaardigheden:

  • Op een agile manier een project aanpakken
  • Rest API
  • Gebruik van Raspberry Pi
  • Front-End via REACT
  • Front-End styling

Project IOT Essentials - weerstation

Beschrijving:

Dit was een van mijn eerste projecten binnen mijn opleiding in het hoger. Het project was volledig zelfstandig en het doel was om een weerstation te maken dat waardes via sensoren kan aflezen en vervolgens naar een Raspberry Pi doorstuurt. De 2 sensoren die bevestigd waren kunnen de temperatuur, hoogte en lichtlevel lezen. Deze waardes moesten vervolgens op een webpagina met grafieken getoond kunnen worden. dit project was volledig in C++ en PHP geschreven. De styling op de webpagina werd gedaan met CSS.

Aangezien er een uitgebreid document en een video over dit document en het feit dit één van mijn eerste projecten was, heb ik veel kunnen leren over het efficiënt documenteren en uitwerken van een volledig project. Dit was het eerste project waar ik een volledig afgewerkt product heb afgeleverd, dus zeker een project om te herinneren.

Mijn bijdrage:

Dit project was volledig zelfstandig. Ik moest dan ook van het begin tot het einde alles zelf opzoeken, testen en verbeteren. Voor deze opdracht had ik nog geen kennis in C++. Om hier dan ook waarden van sensoren mee uit te lezen was dus zeker een uitdaging voor mij in mijn eerste academiejaar.

Vaardigheden:

  • C++/html/css/php
  • Waarden uit sensoren halen
  • Data met grafieken op webpagina tonen in realtime
  • Gericht opzoekwerk
  • Zelfstandig resultaat leveren

Project Hosting - Eigen hosting platform

Beschrijving:

We zijn momenteel nog druk bezig aan dit project. De bedoeling van dit project is dat we een hosting platform moeten opstellen waarbij er PHP-projecten op gehost kunnen worden door een API aan te spreken. Het project moet ook aan een heel extra eisen voldoen. Zo moeten wij bijvoorbeeld het project voldoende beveiligen. Ons groepje van zes personen hebben een heel semester de tijd gekregen door op een agile manier dit project op een succesvole manier te kunnen afronden. Terwijl dit project iets is dat zeer buiten onze comfort zone ligt, gebruiken wij eigenlijk niet veel nieuws. Wij hebben al meerdere keren een API moeten maken, lokaal aan de hand van kubernetes een eigen webstack moeten hosten en systemen beveiligen. Dit project wordt ook extern ondersteund door Tom Cannaerts van het bedrijf iO

Mijn bijdrage:

Aangezien wij nog volop bezig zijn met dit project, moet ik de bijdragen nog zeer beknopt houden. Waar ik me al wel mee heb bezig gehouden is de security van het besturingssysteem. Hier heb ik remote een scapscan op uitgevoerd. Deze scapscan voert dan een testje uit om te kijken hoe veilig het systeem is.

Vaardigheden:

  • Op een agile manier een project aanpakken
  • Rest API
  • Kubernetes
  • Security
  • Automatisering door ansible
  • Linux server

Project API Development - Spotify

Beschrijving:

Dit was mijn eerste project waarbij ik zelf een API heb moeten schrijven. Deze API moest via een webpagina kunnen communiceren. Wij mochten zelf kiezen over welk thema we de API wouden maken. Ik heb ervoor gekozen om dit over Spotify te doen. Eerst was ik van plan om de API van Spotify zelf hier ook in te verwerken, maar heb dit uiteindelijk niet gedaan. De API maakt gebruik van een aantal simpele GET en POST requests. Op de webpagina kan ik bijvoorbeeld een random song opvragen. Of zelf een song toevoegen aan de lijst met songs en hierbij de playlist vermelden waarbij deze bijgezet zou moeten worden.

Mijn bijdrage:

Dit was een project waarbij ik zelf de uitkomst van mocht bepalen. Natuurlijk waren er wel een aantal verplichtingen. Dit project was ook volledig zelfstandig dus alles van het begin tot het einde heb ikzelf gemaakt.

Vaardigheden:

  • Rest API
  • cors security
  • API aanspreekbaar maken in Front-End
  • Front-End styling
